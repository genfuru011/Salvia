# Contributing to Salvia

Thank you for your interest in contributing to Salvia!

## Documentation Guidelines

Salvia maintains documentation in both **English** (`docs/en/`) and **Japanese** (`docs/ja/`).
To prevent documentation drift and ensure a consistent experience for all users, please adhere to the following rules:

1.  **Sync Updates**: If you modify code that affects behavior (e.g., CLI commands, configuration, API), you **MUST** update the corresponding documentation in **BOTH** English and Japanese.
2.  **Check REFERENCE.md**: The `REFERENCE.md` file is the source of truth for usage. Ensure it reflects the latest implementation details (e.g., `deno.json` structure, build process).
3.  **Update CHANGELOG.md**: All notable changes must be documented in `CHANGELOG.md`.
